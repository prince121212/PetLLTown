# 宠物灵魂系统设计

## 核心理念

宠物不是一个"播放器"，而是一个有情绪、有需求、有记忆的生命体。它的行为由内在状态驱动，而不是由用户操作直接触发。用户的操作会影响它的内在状态，内在状态再决定它的外在表现。

## 架构总览

```
用户行为（触摸/语音/离开/回来）
        │
        ▼
┌──────────────────┐
│   事件总线        │  ← 时间事件（每3秒tick/早晚/长时间无交互）
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│   数值系统        │  隐藏数值持续衰减/恢复
│   精力            │  决定是否犯困
│   无聊            │  决定是否主动找事做
│   亲密度          │  决定对用户的热情程度
│   好奇心          │  决定是否做探索性动作
│   压力            │  决定是否需要安抚
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│   心情派生        │  从数值组合推导出当前心情
│   开心            │  默认状态
│   疲倦            │  精力 < 20
│   无聊            │  无聊值 > 70
│   孤独            │  亲密度 < 20 且长时间无交互
│   好奇            │  好奇心 > 70
│   兴奋            │  刚被互动后短暂状态
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│   场景决策引擎    │  根据心情 + 事件 → 选择播放哪个场景组
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│   视频播放器      │  按场景组规则播放（随机选变体）
└──────────────────┘
```

## 数值系统

### 隐藏数值（0-100）

| 数值 | 自然趋势 | 恢复方式 | 影响 |
|------|---------|---------|------|
| 精力 | 缓慢下降（-0.08/秒） | 睡眠时恢复（+0.5/秒） | 低于20触发入睡 |
| 无聊 | 缓慢上升（+0.10/秒） | 用户互动时下降 | 高于70触发主动行为 |
| 亲密度 | 无交互时缓慢下降（-0.03/秒） | 用户互动时上升 | 低于20表现出孤独 |
| 好奇心 | 自然回归中间值 | 新事件发生时上升 | 高于70做探索动作 |
| 压力 | 缓慢自然恢复（-0.02/秒） | 被安抚时快速下降 | 高于60表现焦虑 |

### 数值变化触发

| 用户行为 | 数值变化 |
|---------|---------|
| 按住光球说话 | 无聊 -20, 亲密度 +3, 好奇心 +10 |
| AI 回复完成 | 亲密度 +2, 压力 -5 |
| 长时间不操作（>5分钟） | 无聊 +30, 亲密度 -5 |
| 用户回来（离开>30分钟后） | 好奇心 +30, 亲密度 +5 |
| 触摸宠物（未来功能） | 亲密度 +5, 压力 -3, 无聊 -10 |

## 心情派生规则

优先级从高到低：

1. **疲倦** — 精力 < 20
2. **孤独** — 亲密度 < 20 且最后交互 > 10分钟
3. **无聊** — 无聊值 > 70
4. **好奇** — 好奇心 > 70
5. **兴奋** — 最近5秒内有用户交互
6. **开心** — 默认

## 视频类型与场景组系统

### 两种视频类型

整个系统只有两种视频类型：

**1. 循环视频** — 可以无限重复播放的视频，首尾帧衔接自然
- 待机（idle）：最基础的循环视频，宠物站着/坐着的日常状态
- 睡眠循环（sleep-loop）：宠物趴着睡觉的循环
- 未来可能有：玩耍循环、发呆循环、吃东西循环...

**2. 过渡视频** — 连接两个不同姿态的一次性视频，播放一次后进入下一个状态
- 入睡过渡（sleep-enter）：从站立到趴下闭眼
- 唤醒过渡（sleep-exit）：从趴着到站起来
- 倾听（listening）：从待机姿态到专注听的姿态
- 回应（reply）：做出回应动作后回到待机姿态
- 未来可能有：打哈欠、伸懒腰、摇尾巴...

### 核心规则：不允许打断

**任何视频片段都不允许被打断。** 必须等当前片段播放完毕，才能进入下一段视频。

原因：打断会导致宠物的动作不连贯（上一帧还在趴着，下一帧突然站着），造成"瞬移"。

### 核心规则：不允许跳跃

**不能从一个循环视频直接跳到另一个循环视频。** 两个不同姿态的循环视频之间，必须经过过渡视频。

正确的流转：
```
idle（循环）→ sleep-enter（过渡）→ sleep-loop（循环）→ sleep-exit（过渡）→ idle（循环）
```

错误的流转：
```
idle（循环）→ sleep-loop（循环）  ← 禁止！宠物瞬移了
sleep-loop（循环）→ idle（循环）  ← 禁止！宠物瞬移了
```

### 场景组

场景组是一套配套的视频，包含"进入过渡 + 循环 + 退出过渡"。除了待机组以外，所有循环视频都必须有配套的过渡视频。

```json
{
  "groups": [
    {
      "id": "idle",
      "label": "日常",
      "type": "基础循环",
      "scenes": ["idle"],
      "说明": "最基础的状态，不需要过渡视频，是所有其他组的起点和终点"
    },
    {
      "id": "sleep",
      "label": "睡眠",
      "type": "完整循环组",
      "enter": "sleep-enter",
      "loop": "sleep-loop",
      "exit": "sleep-exit",
      "loopCount": { "min": 2, "max": 6 },
      "说明": "必须走完 enter → loop(n次) → exit 才能回到 idle"
    },
    {
      "id": "interact",
      "label": "交互",
      "type": "过渡序列",
      "scenes": ["listening", "reply"],
      "说明": "listening 和 reply 都是过渡视频，播完自动回到 idle"
    }
  ]
}
```

### 决策队列

当系统决定要切换场景时，不是立即切换，而是把目标放入"待执行队列"：

1. 当前视频片段播放完毕
2. 检查队列中是否有待执行的目标
3. 如果当前在某个组的循环中（如 sleep-loop），且目标需要离开该组：
   - 先播放该组的退出过渡（sleep-exit）
   - 退出过渡播完后，再执行队列中的目标
4. 如果目标需要进入另一个组（如从 idle 进入 sleep）：
   - 先播放该组的进入过渡（sleep-enter）
   - 进入过渡播完后，开始循环

## 场景决策

### 决策时机

决策只在"当前视频片段播放完毕"时发生，不会在播放中途触发。

### 决策优先级（从高到低）

| 优先级 | 条件 | 目标 | 说明 |
|--------|------|------|------|
| 1 | 用户按住了光球 | listening → reply → idle | 如果在睡眠组中，先走 sleep-exit |
| 2 | AI 回复已生成 | reply → idle | 宠物在"说话" |
| 3 | 心情 === 疲倦 | sleep-enter → sleep-loop → sleep-exit → idle | 自然入睡 |
| 4 | 心情 === 兴奋 + 用户刚回来 | 未来：greeting 过渡 → idle | 欢迎主人 |
| 5 | 心情 === 无聊 | 未来：play-start → play-loop → play-end → idle | 自己找乐子 |
| 6 | 默认 | idle 继续循环 | 日常待机 |

## AI 调用策略

### 重要性评分

不是每次都调 AI，只在高重要性事件时才调用：

| 事件 | 重要性 | 是否调 AI |
|------|--------|----------|
| 用户发送语音 | 90 | 是 |
| 用户离开超过30分钟后回来 | 70 | 是（生成欢迎语） |
| 宠物自然入睡 | 10 | 否（纯本地状态机） |
| 宠物无聊做动作 | 5 | 否 |
| 数值 tick | 0 | 否 |

### AI 输出格式

AI 回复时同时输出动作指令：

```json
{
  "reply": "嗯嗯，我在听呢~",
  "emotion": "开心",
  "action": "reply",
  "nextScene": "idle"
}
```

## 时间感知

| 时段 | 行为倾向 |
|------|---------|
| 早上 6-9 点 | 精力恢复快，容易兴奋 |
| 白天 9-18 点 | 正常衰减 |
| 晚上 18-22 点 | 精力衰减加速 |
| 深夜 22-6 点 | 自动进入睡眠组，不响应低优先级事件 |

## 记忆系统

### 记忆类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 对话记忆 | 用户说过的重要的话 | "用户说今天心情不好" |
| 习惯记忆 | 用户的行为模式 | "用户通常晚上9点来" |
| 偏好记忆 | 用户表达过的喜好 | "用户喜欢被叫宝宝" |
| 关系记忆 | 关系里程碑 | "第一次对话"、"连续7天互动" |

### 记忆重要性评分

每条记忆有一个重要性分数（0-1），决定是否保留和是否在 AI 对话时被召回：

- 基础分：0.3
- 强烈情绪（难过/生气/兴奋）：+0.3
- 长文本（>50字）：+0.1
- 关键词命中（爱/喜欢/想/难过/开心/永远/重要/陪）：+0.2
- 时间衰减：每天 -0.02（重要的记忆衰减更慢）

### 记忆召回

AI 对话时，从记忆库中取出重要性最高的 Top 10 条记忆，注入 system prompt，让宠物"记得"之前的对话。

### 关系等级与里程碑

类似 QQ 好友的火花机制，用户和宠物之间有可见的关系标记：

| 里程碑 | 条件 | 标记 | 解锁效果 |
|--------|------|------|---------|
| 初见 | 第一次对话 | 🌱 | 宠物知道你的名字 |
| 小火花 | 连续 2 天互动 | 🔥 | 宠物回复更热情 |
| 大火花 | 连续 7 天互动 | 🔥🔥 | 宠物会主动打招呼 |
| 默契 | 连续 14 天互动 | 💫 | 宠物记住你的习惯 |
| 羁绊 | 连续 30 天互动 | 💎 | 宠物会表达想念 |
| 灵魂伴侣 | 连续 100 天互动 | 👑 | 解锁专属互动动作 |

**断签规则：**
- 连续天数中断后，标记降一级（不会直接清零）
- 每天至少 1 次有效互动（发送语音并收到回复）算"签到"

**关系等级影响：**

| 维度 | 低等级 | 高等级 |
|------|--------|--------|
| AI 回复语气 | 礼貌、简短 | 亲昵、撒娇、更长 |
| 自主行为频率 | 低（安静待着） | 高（主动找你玩） |
| 离开时反应 | 无 | 表现不舍、说再见 |
| 回来时反应 | 普通 | 兴奋、跑过来 |
| 记忆深度 | 只记最近 3 条 | 记住所有重要对话 |

## 离线能力

整个数值系统 + 状态机 + 场景决策都是纯本地逻辑：
- 不依赖网络就能让宠物"活着"
- AI 只在需要生成语言回复时才调用
- 网络断开时宠物照常活动，只是不能"说话"

## 后台管理系统体现

### 场景组管理

后台管理系统中，每个宠物的场景视频应该按"组"展示和管理：

```
蛋挞 管理页面
├── 日常组
│   └── idle（待机）: 2个视频
├── 交互组
│   ├── listening（倾听）: 1个视频
│   └── reply（回应）: 1个视频
└── 睡眠组
    ├── sleep-enter（入睡过渡）: 1个视频
    ├── sleep-loop（睡眠循环）: 1个视频
    └── sleep-exit（唤醒过渡）: 1个视频
```

管理员可以：
- 看到每个组包含哪些场景
- 为每个场景上传多个视频变体
- 组的结构由系统定义，不允许管理员打乱顺序
