# 视频播放状态机

## 核心概念

### 播放队列

系统维护一个**播放队列**（长度 5），提前规划好接下来要播放的视频序列。播放器只需要从队列头部取视频播放，不需要每次播完再临时决策。

```
队列示例（正常待机）：
[idle] → [idle] → [idle] → [idle] → [idle]

队列示例（即将入睡）：
[idle] → [sleep-enter] → [sleep-loop] → [sleep-loop] → [sleep-loop]

队列示例（用户说话触发唤醒）：
[sleep-exit] → [listening] → [reply] → [idle] → [idle]
```

### 事件插入

当事件发生时（用户按光球、精力耗尽等），不是等当前视频播完再处理，而是**立即重排队列**：

- 保留队列第一个（正在播放的，不能打断）
- 清空后面的
- 根据事件和当前状态，重新填充队列

```
事件前队列：[idle(播放中)] → [idle] → [idle] → [idle] → [idle]
用户按光球 →
事件后队列：[idle(播放中)] → [listening] → [reply] → [idle] → [idle]

事件前队列：[sleep-loop(播放中)] → [sleep-loop] → [sleep-loop] → [sleep-exit] → [idle]
用户按光球 →
事件后队列：[sleep-loop(播放中)] → [sleep-exit] → [listening] → [reply] → [idle]
```

### 不打断原则

正在播放的视频（队列第一个）永远不会被打断。事件只影响队列中"还没开始播放"的部分。

## 场景类型

| 类型 | 特征 | 示例 |
|------|------|------|
| 循环场景 | 可以连续重复播放 | idle, sleep-loop |
| 过渡场景 | 只播放一次，连接两个不同姿态 | sleep-enter, sleep-exit, listening, reply |

## 场景组与合法路径

### 合法的场景转换

从任何场景到另一个场景，必须经过合法路径。不允许跳跃。

```
idle ←→ 任何过渡场景（idle 是万能起点/终点）

idle → listening → reply → idle          （交互流程）
idle → sleep-enter → sleep-loop → sleep-exit → idle  （睡眠流程）
```

### 路径规则

1. **从 idle 出发**：可以直接进入任何过渡场景
2. **从 sleep-loop 出发**：必须先经过 sleep-exit 才能到 idle
3. **从 listening 出发**：下一个必须是 listening继续听或者听完了 reply
4. **从 reply 出发**：下一个可以是 idle，也可以是任意从idle到其他循环场景的过渡场景。不能是其他循环场景到idle的过渡场景。这个场景结束的最后一帧就是idle的最后一帧，因此可以视同idle接后续idle可以接的任意场景。
5. **从 sleep-enter 出发**：下一个必须是 sleep-loop
6. **从 sleep-exit 出发**：下一个可以是 idle，也可以是任意从idle到其他循环场景的过渡场景。不能是其他循环场景到idle的过渡场景。这个场景结束的最后一帧就是idle的最后一帧，因此可以视同idle接后续idle可以接的任意场景。

### 路径表（当前场景 → 允许的下一个场景）

| 当前场景 | 允许的下一个 |
|---------|------------|
| idle | idle, listening, sleep-enter, (未来更多过渡场景) |
| listening | 尾帧同idle，因此视同idle |
| reply | 尾帧同idle，因此视同idle |
| sleep-enter | sleep-loop |
| sleep-loop | sleep-loop, sleep-exit |
| sleep-exit | 尾帧同idle，因此视同idle |

## 队列填充策略

### 填充时机

1. **初始化时**：填满 5 个
2. **每次播完一个视频时**：队列头部弹出，尾部补充一个
3. **事件发生时**：保留头部，重排剩余

### 填充逻辑（无事件时的自然填充）

```
function fillQueue(state, queue):
  while queue.length < 5:
    lastInQueue = queue[queue.length - 1]
    next = decideNext(state, lastInQueue)
    queue.push(next)
```

### decideNext（根据最后一个场景决定下一个）

```
function decideNext(state, lastScene):
  // 强制路径（不可选择）
  if lastScene === 'sleep-enter': return 'sleep-loop'
  
  // sleep-loop：根据循环次数决定继续还是醒来
  if lastScene === 'sleep-loop':
    if 已循环次数 < 目标次数: return 'sleep-loop'
    return 'sleep-exit'
  
  // listening：继续听或者回应
  if lastScene === 'listening': return 'reply'
  
  // 尾帧同idle的场景（idle, reply, sleep-exit）：根据心情/数值决定
  // 这些场景后面可以接 idle 能接的任何场景
  if state.mood === '疲倦': return 'sleep-enter'
  // 未来：if state.mood === '无聊': return 'play-start'
  return 'idle'  // 默认继续待机
```

## 事件处理

### 事件类型与响应

| 事件 | 触发条件 | 队列操作 |
|------|---------|---------|
| user_speak | 用户按住光球 | 插入路径到 listening |
| ai_replied | AI 回复生成 | （通常已在队列中，不需要额外操作） |
| energy_low | tick 检测到精力<20 | 插入路径到 sleep-enter |
| energy_full | 睡眠中精力恢复满 | 插入 sleep-exit |

### 事件插入逻辑

```
function handleEvent(event, state, queue):
  playing = queue[0]  // 正在播放的，不动
  
  // 计算从当前播放场景到目标的合法路径
  target = eventToTarget(event)
  path = findPath(playing, target)
  
  // 重建队列
  queue = [playing] + path + fillRemaining(state, path.last)
  queue = queue.slice(0, 5)
```

### findPath（找到合法路径）

```
// 尾帧同idle的场景，可以直接接idle能接的任何场景
idleLikeScenes = ['idle', 'listening', 'reply', 'sleep-exit']

function findPath(from, to):
  // 如果 from 是尾帧同idle的场景，可以直接到任何idle能接的目标
  if from in idleLikeScenes:
    return [to]
  
  // 如果 from 在 sleep-loop，需要先 exit
  if from === 'sleep-loop':
    if to === 'sleep-loop': return [to]
    if to === 'sleep-exit': return [to]
    return ['sleep-exit', to]  // sleep-exit 尾帧同idle，可以直接接目标
  
  // 如果 from 是 sleep-enter，必须走完 sleep-loop → sleep-exit
  if from === 'sleep-enter':
    return ['sleep-loop', 'sleep-exit', to]

function pathFromIdle(to):
  // idle（及尾帧同idle的场景）可以直接到这些目标
  if to === 'listening': return ['listening']
  if to === 'sleep-enter': return ['sleep-enter']
  // 未来更多...
  return ['idle']
```

## 视频选择

每个场景可能有多个视频变体（videoUrls 数组）。从队列取出场景 ID 后：

```
function pickVideo(sceneId, manifest):
  action = manifest.actions.find(a => a.id === sceneId)
  if action.videoUrls.length === 0: return fallbackVideo
  return action.videoUrls[random(0, action.videoUrls.length - 1)]
```

## 预加载

队列中第 2 个视频（下一个要播的）会被提前用第二个 decoder 加载好，实现无缝切换。

## 完整流程图

```
┌─────────────────────────────────────────────────────────┐
│                    播放队列 (长度5)                        │
│  [当前播放] → [下一个(预加载)] → [3] → [4] → [5]         │
└────────┬────────────────────────────────────────────────┘
         │
         │ 当前视频播完
         ▼
┌─────────────────────┐
│ 弹出队列头部          │
│ 尾部补充一个新场景    │
│ 预加载新的第2个       │
│ 开始播放新的头部      │
└─────────────────────┘

         │ 事件发生（用户按光球/精力耗尽/...）
         ▼
┌─────────────────────┐
│ 保留队列头部(不打断)  │
│ 清空 2-5 位          │
│ 计算合法路径到目标    │
│ 重新填充队列         │
│ 预加载新的第2个       │
└─────────────────────┘
```

## 示例场景

### 场景 1：正常待机，用户说话

```
初始队列：[idle①] → [idle②] → [idle③] → [idle④] → [idle⑤]
                     ↑ 正在播放 idle①

用户按光球 → 事件 user_speak
当前 idle① 不打断，重排后面：
新队列：  [idle①] → [listening] → [reply] → [idle] → [idle]
                     ↑ idle① 播完后立即进入 listening
```

### 场景 2：睡眠中，用户说话

```
初始队列：[sleep-loop①] → [sleep-loop②] → [sleep-loop③] → [sleep-exit] → [idle]
                           ↑ 正在播放 sleep-loop①

用户按光球 → 事件 user_speak
当前 sleep-loop① 不打断，需要先 exit 再 listen（sleep-exit 尾帧同idle，可直接接 listening）：
新队列：  [sleep-loop①] → [sleep-exit] → [listening] → [reply] → [idle]
```

### 场景 3：待机中，精力耗尽

```
初始队列：[idle①] → [idle②] → [idle③] → [idle④] → [idle⑤]
                     ↑ tick 检测到精力 < 20

事件 energy_low：
新队列：  [idle①] → [sleep-enter] → [sleep-loop] → [sleep-loop] → [sleep-loop]
```

### 场景 4：睡眠中，精力恢复满

```
初始队列：[sleep-loop①] → [sleep-loop②] → [sleep-loop③] → [sleep-loop④] → [sleep-loop⑤]
                           ↑ tick 检测到精力 >= 80

事件 energy_full：
新队列：  [sleep-loop①] → [sleep-exit] → [idle] → [idle] → [idle]
```
