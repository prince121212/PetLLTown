# 素材交付规范

本文档用于约束后续进入本项目的宠物透明视频素材。目标是确保素材能稳定转成小程序可播放的透明背景视频，并避免再次出现“看起来抠图了，但实际没有 alpha”的问题。

## 1. 首选交付规格

后续优先交付这种源素材：

```text
格式：WebM
编码：VP9
透明：必须带真实 alpha 通道
尺寸：720x960
比例：3:4 竖版
帧率：24fps
时长：建议 8-12 秒
音频：不要音频
背景：透明背景，不要黑底、白底、绿幕底
循环：首尾动作尽量自然衔接
主体：宠物完整可见，不裁耳朵、尾巴、脚
```

合格示例：

```text
生成布偶猫视频_transparent_fixed.webm
VP9 / 720x960 / 24fps / ALPHA_MODE=1
```

## 2. 可接受的备选规格

如果无法交付 VP9 WebM，可以交付以下任一种，但必须真的包含 alpha：

- ProRes 4444 MOV。
- PNG 序列，单帧必须是 RGBA。
- 其他明确带 alpha 的视频格式，例如像素格式包含 `rgba`、`yuva`、`bgra`、`gbrap`。

不接受只写着“Alpha”但实际没有透明通道的视频。

## 3. 明确不接受的素材

以下素材不能直接作为透明源素材使用：

- 普通 H.264 MP4：通常是 `h264 + yuv420p`，没有 alpha。
- 普通 HEVC MP4：如果是 `hevc Main + yuv420p`，没有 alpha。
- 黑底视频：背景是黑色像素，不等于透明。
- 白底视频：背景是白色像素，不等于透明。
- 绿幕视频：需要额外抠像，不算合格透明素材。
- 只有静态封面透明、视频帧不透明的文件。
- 分辨率过小、主体被裁切、动作不循环的素材。

这类素材如果必须使用，需要先重新抠像或重新生成透明版本。

## 4. 验收命令

收到素材后，先用 `ffprobe` 检查：

```bash
ffprobe -hide_banner -show_streams -show_format -print_format json "/path/to/source-video"
```

合格判断：

- WebM 源素材应看到 `codec_name=vp9`。
- WebM 源素材应看到 `ALPHA_MODE=1`。
- 尺寸应是 `720x960`。
- 帧率建议是 `24/1`。

然后抽取 alpha 验证：

```bash
ffmpeg -hide_banner -y \
  -c:v libvpx-vp9 \
  -i "/path/to/source-video.webm" \
  -frames:v 1 \
  -vf alphaextract \
  /tmp/petllt-alpha-check.png
```

再检查黑白范围：

```bash
ffmpeg -hide_banner \
  -i /tmp/petllt-alpha-check.png \
  -vf signalstats,metadata=print:file=- \
  -frames:v 1 \
  -f null -
```

合格判断：

- `YMIN` 应接近 `0`，代表背景透明。
- `YMAX` 应接近 `255`，代表主体不透明。
- 如果整张图几乎全黑、全白或抽取失败，都不合格。

## 5. 转入小程序前的运行时规格

小程序最终不直接播放 alpha WebM，而是统一转成双通道 MP4：

```text
格式：MP4
编码：H.264
像素格式：yuv420p
尺寸：1440x960
帧率：24fps
音频：无
布局：左右双通道
左半边：720x960 RGB 画面
右半边：720x960 alpha mask
```

也就是说，源素材是：

```text
720x960 透明视频
```

运行时素材是：

```text
1440x960 双通道 MP4
```

## 6. 推荐命名规则

源素材命名：

```text
{petId}-{action}-transparent-src.webm
```

运行时双通道 MP4 命名：

```text
{petId}-{action}-alpha-pack-h.mp4
```

示例：

```text
xiaotuanzi-idle-transparent-src.webm
xiaotuanzi-idle-alpha-pack-h.mp4
```

## 7. 视觉安全区

为了适配首页主舞台，素材画面需要遵守：

- 宠物主体居中。
- 耳朵、尾巴、脚不要贴边。
- 顶部至少留出 5% 安全边距。
- 左右至少留出 5% 安全边距。
- 底部可以接近地面，但脚和尾巴不能被裁掉。
- 不要把阴影、文字、水印做进透明视频里，除非明确需要。

## 8. 交付清单

每次交付新宠物或新动作，至少提供：

- 源透明视频：优先 WebM VP9 alpha。
- 首帧或代表帧 PNG：用于人工快速预览。
- 素材说明：宠物 ID、动作名、时长、是否可循环。

交付后必须通过：

- `ffprobe` 检查。
- `alphaextract` 检查。
- 双通道 MP4 右半边 alpha mask 检查。
- 小程序真机或开发者工具截图检查。

## 9. 一句话标准

以后给这个项目的主宠物动画素材，默认按这个标准给：

```text
720x960、24fps、8-12 秒、VP9 WebM、真实 alpha、无音频、透明背景、主体完整、首尾可循环。
```
